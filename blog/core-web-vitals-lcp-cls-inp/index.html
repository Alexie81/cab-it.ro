Core Web Vitals: LCP, CLS și INP, explicate simplu

Cei trei indicatori descriu viteza percepută, stabilitatea vizuală și răspunsul paginii la interacțiunile utilizatorului.

Blog / Core Web Vitals

Ce măsoară fiecare indicator

LCP măsoară timpul până când cel mai mare element relevant din zona vizibilă este afișat. De regulă, acesta este imaginea principală sau un bloc mare de text. CLS cumulează deplasările neașteptate ale elementelor, precum butoane sau imagini care își schimbă poziția în timpul încărcării. INP măsoară cât de repede răspunde pagina la clickuri, atingeri și tastare.

Țintele uzuale pentru o experiență bună sunt LCP sub 2,5 secunde, CLS sub 0,1 și INP sub 200 ms, evaluate la percentila 75 din date reale.

Ce optimizezi pentru LCP și CLS

  • comprimă imaginea principală și folosește dimensiunea potrivită dispozitivului;
  • încarcă prioritar resursa LCP și nu aplica lazy loading imaginii din hero;
  • setează lățimea și înălțimea imaginilor, video-urilor și embed-urilor;
  • reduce CSS-ul care blochează primul paint și numărul de fonturi;
  • rezervă spațiu pentru bannere, formulare și componente încărcate dinamic.

Lazy loading este util sub fold, dar poate întârzia elementul principal dacă este aplicat fără discernământ.

Cum îmbunătățești INP

Un INP slab apare când firul principal este ocupat cu JavaScript, randări complexe sau callback-uri lungi. Elimină bibliotecile nefolosite, amână funcțiile care nu sunt necesare imediat și fragmentează sarcinile lungi. Testează meniurile, formularele și componentele interactive pe telefoane mai lente, nu doar pe desktop.

Un scor bun într-un test de laborator este util pentru diagnostic, dar nu înlocuiește datele reale. Hostingul, cache-ul, conexiunea și dispozitivele vizitatorilor influențează rezultatul final.

Ordinea practică

  1. identifică elementul LCP pe mobil;
  2. rezolvă resursele blocate și imaginile mari;
  3. elimină deplasările de layout;
  4. profilează interacțiunile lente;
  5. verifică din nou în date reale.
